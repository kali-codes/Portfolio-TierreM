PlutoPay

Problem

Users need a way to quickly pay for items, make bill payments and save money without the use of their physical wallets.

Solution

The solution is to create an electronic wallet app with key features in order to gain convenience.

My Roles

  • UX Researcher
  • UX Designer
  • UI Designer

Project Phases

  • Research
  • Concepting
  • Prototyping and User Testing
  • High Fidelity Wireframes

Competitive Analysis

The first step is a competitive analysis. Keeping the problem in mind, I completed a search for competitors of the app PlutoPay The two apps that I found were Apple Wallet and Folio: Wallet App.

User Research

I then conducted 3 remote interviews with the goal to:

  • Determine what payment method a user is currently using.
  • Better understand what would motivate a user to rely on an electronic wallet app.
  • Determine which features users would utilize using an electronic wallet app.

I then drew up a list of insights and created an affinity map.

Insights:

  • All users mainly use their debit card for purchases.
  • All users made it known that they will not use an electronic wallet app if the app has any monthly subscriptions or if the app makes unauthorized purchases.
  • The main feature users highlighted was a security system within the app. That being a pin, passcode, or 2 step authentication.
  • The convenience of an electronic wallet app is what will motivate the users to use it.

User Personas

After careful consideration of insights found from user research, I created two personas. David and Andrea.

User Flows

David and Andrea now have journeys that need to be defined to follow their goals of checking their spending habits and paying bills.

Card Sorting

An open card sorting study was made using the website optimalworkshop.com in order to test the effectiveness of the sitemap.

The study was sent to 10 participants and 5 completed the study.

Card Sorting Results

I chose for this study to be open in order to get more ideas for the sitemap and see how others would group the cards given.

Doing this study definitely made me reaarrange a few cards in my sitemap and even add more cards to create a more effective and efficient sitemap.
